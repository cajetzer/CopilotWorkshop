## üéØ Your Exercises

### Exercise 8.8: From Logs to Issue ‚Äî Clarity from Chaos ‚≠ê *You lead this one*

**Your role**: Transform raw bug reports into structured, actionable issues  
**Time**: 10 minutes  
**[View full exercise ‚Üí](../EXERCISES.md#exercise-88-from-logs-to-issue--clarity-from-chaos)**

**What you'll create:**
- Structured issue from chaotic input
- Reproducible workflow for issue triage
- Consistent quality regardless of input quality

**Your "before" pain:**
Traditional bug report workflow:
1. Receive vague report with wall of logs
2. Spend 15-20 minutes parsing logs, finding the actual error
3. Write structured issue with reproduction steps
4. Add labels and route to correct team
5. Sometimes miss key details, requiring follow-up questions

**Time per issue**: 20-25 minutes  
**Triage throughput**: 2-3 issues per hour  
**Quality**: Varies (when rushed, details get missed)  
**Mental cost**: Draining (tedious log parsing)

**Your "after" win:**
With Copilot-assisted triage:
1. Paste raw description and logs into GitHub issue
2. Ask Copilot to structure it (clear title, summary, reproduction steps, relevant log excerpts)
3. Review and refine
4. Apply labels

**Time per issue**: 5 minutes  
**Triage throughput**: 10-12 issues per hour (4x improvement)  
**Quality**: Consistent (AI doesn't get fatigued)  
**Mental cost**: Minimal (focus on validation, not parsing)

**Per sprint** (10 bug reports): 2.5-3 hours saved  
**Per year** (26 sprints): 65-78 hours = 1.5+ weeks of recovered time

**Your validation moment:**
> *"Issue triage used to be a bottleneck‚Äî20 minutes per bug report adds up fast. Now I can process 4x more issues in the same time without sacrificing quality. That's not just convenient‚Äîthat's measurable workflow improvement. My expertise in writing clear issues now scales through AI."*
